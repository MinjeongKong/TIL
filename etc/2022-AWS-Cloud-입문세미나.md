# On-Premise
클라우드 컴퓨팅 이전, IT자원을 직접 구매, 구축, 유지보수, 권한 관리 하는 인프라 관리 방식. 실제 물리적인 서버를 사용. <br>
계획, 자본금, 공간, 물리적 보안 등 여러 고려사항이 있음.

# Cloud Computing
- 큰 초기 투자 비용 없이 인터넷을 통해 유연한 IT 리소스와 애플리케이션을 필요할 때 언제든(on-demand) 사용한 만큼만 요금을 내고 이용하는 서비스(pay-as-you-go)

# Amazon Web Service
전 세계적으로 분포한 데이터 센터에서 200개가 넘는 완벽한 기능의 서비스를 제공하는, 세계적으로 가장 포괄적이고 널리 채택되고 있는 클라우드 플랫폼.

<br>

# 클라우드 컴퓨팅의 장점
컴퓨팅 리소스를 만들 때 초기 투자비용이 필요한 On-Premise 방식은 예측과 계획을 통해 선투자해 사용. 예측이 정확하지 않으면 서비스 지연, 비용 손실 등이 발생할 수 있음.

## 1. 자본 비용을 가변 비용으로 대체
하드웨어 구매 또는 데이터 센터 구축 불필요
- 리소스를 사용하는 만큼 비용 지불
- 초기 자본 비용 절감

## 2. 규모의 경제로 얻게 되는 이점
큰 규모의 경제를 활용
- 자체 보유보다 저렴한 비용
    - 전문화된 하드웨어 및 소프트웨어
    - 대규모 하드웨어 구입

## 3. 용량 추정 불필요
- 필요에 따라 확장 및 축소
- 오버프로비저닝 불필요 (오토스케일링을 통해 용량 낭비 방지, 서버의 규모가 자동으로 확장,축소)

## 4. 속도 및 민첩성 개선
하드웨어를 설치 및 설정할 때까지 기다릴 필요가 없음
- 한 번의 클릭으로 새 IT 리소스 확보
- 리소스 개발 시간 단축

## 5. 중요한 문제에 집중
인프라가 아니라 애플리케이션에 집중
- 리소스를 확보하여 새 프로젝트에 투자
- 데이터 센터 운영 및 유지 관리에 비용 투자 불필요
- 일회용 리소스를 통해 신속한 실험 가능

## 6. 몇 분 만에 전 세계에 배포
몇 분 만에 전 세계에 배포
- 전 세계에 분포된 여러 AWS 리전
- 애플리케이션을 사용자와 가까이 유지
- 고가용성 및 재해 복구 촉진

### **On-Premise 방식일 경우**
1. 현지 법인 설립
2. 사무실 대여
3. 서버실 대여
4. 장비 구입
5. 네트워크 환경 구현
6. 서버 설치
7. 서비스 설정
8. 테스트
9. 서비스 시작

클라우드 사용시 3~9 단계가 간략해짐
